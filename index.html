Intermediate System Design — From Foundations to Production

This book takes a software engineer who has completed a beginner system-design course and guides them to a practical, applied level. You will design service boundaries, choose data and storage patterns for scale, define SLOs/SLIs, implement CI/CD and observability, and deliver a capstone prototype with fault-injection tests and a blameless postmortem.

Who this is for

You have beginner-level system design vocabulary (can name core components and draw simple request flows) and want to make production-grade decisions: service boundaries, data/storage choices, SLOs and error budgets, safe deploys, and operational readiness.

What you will build

A deployable, observable prototype for a domain you choose (e.g., e‑commerce catalog, metrics pipeline, or chat). You will define NFRs, implement CI/CD, ship to a realistic runtime (managed Kubernetes or serverless), run a chaos experiment, and publish a postmortem.

How to use this book

  • Cadence: Suggested 12–16 weeks. Each week: read one section, do one applied exercise, and capture decisions in a design log.
  • Paired learning: Every design choice is followed by a small implementation task to solidify trade‑offs.
  • Reviews: Schedule three design reviews (after Chapters 3, 5, and 8) and a final capstone review.
  • Evidence: Maintain SLO dashboards, runbooks, and postmortems. Treat this as a miniature production program.
Tip: Keep a shared Design Log (decisions, alternatives, risks). It powers faster incident response and better postmortems later.

Course roadmap & timeframe

Plan for 12–16 weeks. Milestones (align with Chapter 9 rubric):

  1. Week 0: Choose capstone domain; define baseline NFRs and initial SLOs.
  2. Weeks 1–3: Components, data model, and API contracts.
  3. Weeks 3–6: Core services implemented; CI/CD green.
  4. Weeks 6–9: Deploy; add observability; run load/chaos tests.
  5. Weeks 9–12: Finalize dashboards, runbooks, postmortem, and architecture review.
