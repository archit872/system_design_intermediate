Glossary of Terms

Definitions follow a consistent pattern: Plain → Formal → Pitfall → Example. Where useful, terms link to related chapters for deeper study.

A

ABAC (Attribute-Based Access Control)

Plain: Permissions decided by attributes (user role, time, location), not just fixed roles.

Formal: Policy evaluation over subject, object, action, and environment attributes.

Pitfall: Policy sprawl; hard to audit without tooling.

Example: “Analysts can view reports only 09:00–18:00 IST from corp IPs.” (See Ch.7)

ACID

Plain: Strong transaction guarantees in relational DBs.

Formal: Atomicity, Consistency, Isolation, Durability.

Pitfall: High contention + strict isolation can tank throughput.

Example: Money transfer debits and credits either both happen or none. (Ch.4)

API Gateway

Plain: Front door for all APIs—auth, rate limits, routing.

Formal: L7 mediator adding cross-cutting policies and observability.

Pitfall: Becoming a monolith of logic instead of policy.

Example: JWT verification + quota per API key. (Ch.3, Ch.6)

C

CAP Theorem

Plain: In a partition, you must prefer Consistency or Availability.

Formal: No distributed system can simultaneously guarantee C, A, and P.

Pitfall: Treating CAP as a design recipe for normal operations.

Example: Dynamo favors A; Spanner favors C with global time. (Ch.4–5)

CDN (Content Delivery Network)

Plain: Caches content near users to cut latency.

Formal: Edge PoPs serve cacheable objects with configurable TTL and validation.

Pitfall: Forgetting cache keys/headers → stale or missed hits.

Example: Cache-Control + ETag for product pages. (Ch.5)

Circuit Breaker

Plain: Fail fast after repeated errors to protect systems.

Formal: Closed → Open → Half-open state machine guarding calls.

Pitfall: Never resetting thresholds; flapping.

Example: Open breaker for slow recommendations; serve baseline. (Ch.3, Ch.5)

Consistent Hashing

Plain: Spread keys across nodes so rebalancing moves few keys.

Formal: Hash ring with virtual nodes to minimize key movement on membership change.

Pitfall: Skew without enough virtual nodes.

Example: Cache cluster with 200 vnodes per host. (Ch.5)

CQRS

Plain: Separate write model from read models.

Formal: Commands mutate authoritative state; queries read denormalized projections.

Pitfall: Ceremony without need; rebuild complexity.

Example: Orders write to SQL; analytics reads from pre-aggregated store. (Ch.6)

CRDT

Plain: Data types that merge without conflicts across replicas.

Formal: Commutative/associative/idem. semilattices with monotone join.

Pitfall: Hidden semantics; tombstone bloat.

Example: Counter that merges offline edits on sync. (Ch.10)

D

DLQ (Dead Letter Queue)

Plain: Parking lot for messages that can’t be processed.

Formal: Terminal retry destination with retention for manual/automated reprocessing.

Pitfall: Letting DLQs fill without alerts.

Example: Poison message with bad schema routed to DLQ. (Ch.3)

Denormalization

Plain: Duplicate data to speed reads.

Formal: Intentional redundancy to optimize query patterns.

Pitfall: Complex writes and drift without backfills.

Example: Precomputed counts table for dashboards. (Ch.4)

Distributed Tracing

Plain: Follow a request across services.

Formal: Trace with spans, context propagation (e.g., W3C traceparent).

Pitfall: Missing propagation at gateways.

Example: Trace shows slow DB span → add index. (Ch.8)

E

Event Sourcing

Plain: Store events, derive current state by replay.

Formal: Append-only event log + projections; snapshots bound replay time.

Pitfall: Changing past events; slow rebuilds without snapshots.

Example: Order lifecycle events rebuild order state. (Ch.6)

H

Hedged Requests

Plain: Send a backup request after a threshold to cut tail latency.

Formal: Time-based speculative execution with cancellation of losers.

Pitfall: Doubling load; needs caps and adaptive thresholds.

Example: Issue a second read after p95 latency elapses. (Ch.3, Ch.5)

I

Idempotency

Plain: Doing an action twice has the same effect as once.

Formal: f(f(x)) = f(x) for requests with the same key/intent.

Pitfall: Forgetting unique keys; duplicate side effects.

Example: Create-payment with Idempotency-Key header. (Ch.3, Ch.5)

J

Jitter

Plain: Randomness in backoff/expiry times.

Formal: Randomized delay to decorrelate client behavior.

Pitfall: Synchronised retries causing herds.

Example: 50% jitter on 200ms backoff. (Ch.3, Ch.5)

L

Load Balancer (L4/L7)

Plain: Shares traffic across instances.

Formal: L4 (TCP/UDP) vs. L7 (HTTP-aware) routing with health checks.

Pitfall: Sticky sessions hide statefulness; hurts scaling.

Example: Envoy routing by path and header. (Ch.3)

M

Microservices

Plain: Many small services with clear contracts.

Formal: Independently deployable bounded contexts with network isolation.

Pitfall: Ops overhead > benefits for small teams.

Example: Payments split from monolith; gateway handles auth. (Ch.6)

Modular Monolith

Plain: One deploy with strict internal boundaries.

Formal: Enforced module APIs and ownership within a single runtime.

Pitfall: Sloppy boundaries → “big ball of mud.”

Example: Catalog/Cart/Checkout modules in one app. (Ch.6)

O

OAuth 2.0

Plain: Let an app access your data without giving it your password.

Formal: Authorization framework with grants (code, client credentials, etc.).

Pitfall: Long-lived tokens and overly broad scopes.

Example: Mobile app gets access token for profile API. (Ch.7)

OIDC (OpenID Connect)

Plain: Identity layer on top of OAuth2.

Formal: Standardized ID token (JWT) that asserts who the user is.

Pitfall: Confusing ID tokens with access tokens.

Example: Verify user identity at gateway. (Ch.7)

Q

Quorum

Plain: Majority agreement before accepting reads/writes.

Formal: Choose R, W with R+W > N to guarantee intersecting replicas.

Pitfall: Latency spikes when quorum is high under failure.

Example: N=5, W=3, R=2 for analytics store. (Ch.5)

R

Rate Limiting (Token Bucket)

Plain: Control how fast clients can call you.

Formal: Bucket with capacity C and refill rate r; requests consume tokens.

Pitfall: Global limits without per-tenant fairness.

Example: 100 req/s burst, 20 req/s sustained per API key. (Ch.5)

RED / USE

Plain: Monitoring checklists: user path vs. resource health.

Formal: RED = Rate, Errors, Duration; USE = Utilization, Saturation, Errors.

Pitfall: Alerting on low-value causes rather than symptoms.

Example: p95 latency + queue depth dashboards. (Ch.8)

Replication (Leader/Follower, Multi-Leader)

Plain: Keep copies of data for durability and scale.

Formal: Single-leader, multi-leader, or leaderless quorum protocols.

Pitfall: Conflict resolution and lag ignored.

Example: Read replicas for product catalog. (Ch.4–5)

Retry with Exponential Backoff

Plain: Try again later with growing delays.

Formal: delay = base × factor^n ± jitter; cap attempts and total budget.

Pitfall: Infinite retries causing storms.

Example: 3 attempts, 50→100→200ms with 20% jitter. (Ch.3)

S

Saga

Plain: Break a big transaction into steps with compensations.

Formal: Orchestrated/choreographed sequence with failure compensation.

Pitfall: Missing compensations on partial failure.

Example: Reserve inventory → charge card → on fail, release inventory. (Ch.6)

Sharding (Partitioning)

Plain: Split a big dataset across machines.

Formal: Range/hash/geo partitions with rebalancing and routing.

Pitfall: Premature sharding; cross-shard joins.

Example: Hash by user ID for messages. (Ch.4)

SLA / SLO / SLI

Plain: Promise, target, and measurement of service quality.

Formal: SLAs contractual; SLOs internal targets; SLIs metrics.

Pitfall: Vanity metrics; unclear user journeys.

Example: SLO 99.9% availability; SLI based on success rate <200ms. (Ch.5, Ch.8)

Stale-While-Revalidate (SWR)

Plain: Serve stale cache while refreshing in background.

Formal: HTTP directive allowing temporary stale responses on revalidation.

Pitfall: Unbounded staleness without caps.

Example: 60s SWR for celebrity profile to survive spikes. (Ch.5)

T

Timeout

Plain: Give up waiting after a set time.

Formal: Upper bound on synchronous wait; must align with user latency budget.

Pitfall: No timeouts → hung threads, cascading failures.

Example: 200ms per hop with overall 600ms budget. (Ch.3)

TLS

Plain: Encrypts data in transit.

Formal: TLS 1.2/1.3 with strong ciphers; mTLS for service-to-service auth.

Pitfall: Mixed-mode plaintext inside perimeter; weak cert rotation.

Example: mTLS between gateway and services. (Ch.7)

Token Bucket

Plain: Rate limit that allows bursts.

Formal: Bucket capacity B refilled at rate r; request consumes one token.

Pitfall: Shared bucket causes noisy neighbor.

Example: Per-tenant buckets at gateway. (Ch.5)

V–Z

Versioning (APIs/Events)

Plain: Change interfaces without breaking clients.

Formal: Backward-compatible schemas; additive fields; deprecation windows.

Pitfall: Breaking changes without migration plan.

Example: Add field with default; keep old consumers running. (Ch.6)
