Appendix

Pitfalls, Checklists & Progression

This appendix distills lessons into a set of pitfalls to avoid, a review checklist for design work, and a progression map for advancing from beginner to advanced system design competence.

Top 10 Pitfalls & Fixes

  1. Over-engineering early → Fix: Start simple, add only under proven load.
  2. Premature sharding → Fix: Scale vertically/read replicas first; shard when node limits reached.
  3. Missing SLOs → Fix: Define SLIs + SLOs before debating architecture.
  4. No observability → Fix: Instrument RED/USE metrics, logs, and traces from day one.
  5. Retry storms → Fix: Exponential backoff + jitter, idempotency, bounded retries.
  6. Ignoring data privacy → Fix: Apply data minimization, encryption, and compliance flows.
  7. One-size-fits-all DB → Fix: Match store to workload; evaluate specialized stores carefully.
  8. Microservices dogma → Fix: Modular monolith → selective service extraction.
  9. No rollback plan → Fix: Feature flags, canaries, expand–contract migrations.
  10. Skipping postmortems → Fix: Make them blameless, actionable, and required.

System Design Review Checklist

  • Requirements clear? Functional + non-functional, with SLIs/SLOs.
  • Data model and storage choice justified? Scaling strategy explicit?
  • Communication patterns defined? Sync/async trade-offs noted?
  • Reliability features: retries, idempotency, failover, backpressure.
  • Security: authN/authZ, encryption, privacy, compliance hooks.
  • Observability: RED/USE metrics, logs, traces, alerting tied to SLOs.
  • Trade-offs documented? At least two options considered per major decision.
  • Failure scenarios explored? Rollback and recovery paths defined.
  • Operational load measured? Runbooks, automation, toil caps.
  • Progression plan: what evolves with growth, what stays simple now?
Usage: Bring this checklist to every design review; iterate until all boxes ticked.

Progression Map

Beginner End Goals

  • Core vocabulary (client, server, API, DB).
  • Basic workflows (CRUD, cache, request flow).
  • Lightweight case studies (URL shortener, feed).

Intermediate End Goals

  • Applied competence in mid-scale designs.
  • Ability to evaluate trade-offs with diagrams.
  • Capstone project with end-to-end architecture.

Advanced Entry Point

  • Multi-region consistency (consensus, Raft, Paxos).
  • Formal models (TLA+, model checking).
  • Cost-aware scaling and distributed algorithms.
Tip: Use your capstone project as a springboard into advanced topics—add multi-region, model a consensus log, or simulate chaos.
